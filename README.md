# Local RAG Chatbot with Qwen on CPU

A local Retrieval-Augmented Generation (RAG) chatbot for answering questions about the National Bank of Cambodia (NBC).

The planned chatbot will retrieve relevant information from the local NBC dataset and use a Qwen language model running locally on the CPU to generate a grounded answer. No cloud LLM API is required for the RAG workflow.

## Goals

- Run the chatbot locally with a Qwen model on CPU.
- Retrieve relevant NBC information before generating an answer.
- Reduce hallucinations by grounding responses in the local knowledge base.
- Keep the project simple enough to run for development and educational use.
- Make it possible to replace or expand the dataset without changing the chatbot interface.

## RAG workflow

```text
User question
     |
     v
Question embedding
     |
     v
Local vector search over NBC knowledge
     |
     v
Retrieved context + question
     |
     v
Qwen running on CPU
     |
     v
Grounded chatbot answer
```

The expected pipeline is:

1. Load the Q&A records from `src/data/nbc_data.json`.
2. Convert each record into searchable text.
3. Create and persist local embeddings in a vector index.
4. Retrieve the most relevant records for each user question.
5. Add the retrieved records to the prompt.
6. Generate the final response with a local Qwen model.

## Dataset

`src/data/nbc_data.json` contains curated question-and-answer records about the National Bank of Cambodia, including:

- NBC history and general information
- Mandate and functions
- Monetary policy
- Banking supervision
- Currency and the Riel
- Payment systems
- Financial stability
- International relations
- Financial inclusion
- Digital economy and fintech

The dataset is intended for educational and testing purposes. It should not be treated as official financial, legal, or investment advice. For authoritative information, consult official NBC publications and channels.

## Requirements

- Python 3.14 or newer, as specified in `pyproject.toml`
- `uv` for environment and dependency management
- A CPU with enough memory for the selected Qwen model
- Additional model, embedding, vector-search, and web dependencies once those components are implemented

## Installation

Clone the project and enter its directory:

```bash
git clone <repository-url>
cd NBC-BOT
```

Create the environment and install the currently defined dependencies:

```bash
uv sync
```

Activate the virtual environment if desired:

```bash
# Windows PowerShell
.venv\Scripts\Activate.ps1

# macOS/Linux
source .venv/bin/activate
```

## Running the current project

At the moment, the entry point is only a starter placeholder:

```bash
uv run python main.py
```

It currently prints:

```text
Hello from nbc-bot!
```

## Configuration

Local secrets and machine-specific settings should be stored in `.env`. The file is ignored by Git. The chatbot is intended to work without an API key when using fully local models.
